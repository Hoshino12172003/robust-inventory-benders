# Secondary Cut Selection Redesign

## Summary

This change redesigns relative cut selection so that it cannot suppress the primary worst-case Benders cut. A primary cut with a feasible incumbent and positive numerical violation is always added, subject only to the existing duplicate and not-violated checks. Threshold screening now applies only to additional cuts generated by no-good exclusions.

## Implementation

- Added explicit primary and secondary cut roles in the Benders iteration.
- Kept the primary cut independent of absolute and relative cut thresholds.
- Applied static or adaptive relative screening only to secondary multi-cuts.
- Added an adaptive secondary threshold using the current master cut count, cumulative master-time share, recent master runtime, global Benders gap, and final exact phase.
- Retained all positively violated secondary cuts while the master is small.
- Restored all positively violated secondary cuts during the final exact phase.
- Preserved duplicate-cut and numerical not-violated checks.
- Recorded each secondary cut's normalized violation, marginal normalized violation relative to the primary cut, add decision, skip reason, and active threshold.
- Added secondary cut totals and adaptive-policy settings to result metadata and experiment CSV output.

## Mathematical Validity

- The robust model and robust dual MILP are unchanged.
- Every cut is generated from a feasible incumbent, never from an objective bound.
- The target-Gamma robust objective bound remains the only robust cost used for a conservative Benders upper bound.
- Objective bounds from no-good-constrained additional solves never update the unconstrained robust upper bound.
- Benders termination and upper-bound validity rules are unchanged.

The configuration is provided for local execution only. No evaluation seeds were run in this task. `selected_algorithm_parameters.yaml` remains pending; it now includes null placeholders for all adaptive secondary-cut policy fields so a future selected policy can be reproduced exactly.

## Review Fixes

- Replaced the ratio-based secondary score with an RHS-based marginal normalized violation.
- The marginal baseline is `max(theta_current, primary_rhs)`, so a secondary cut weaker than or equal to the primary cut has zero local strengthening value.
- Secondary relative selection and stall ranking now use the same RHS-based marginal measure.
- Added the four adaptive secondary-cut policy fields to final selected-parameter loading.
- Added strict type and range validation for the selected policy.
- Explicitly disabled adaptive secondary selection for `standard_benders` and `static_inexact_benders`.
- Preserved the primary-cut rule, robust model, UB validity rules, no-good formulation, and termination logic.
